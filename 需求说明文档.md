
# 一、卧龙决策支持系统 产品需求说明书（PRD v1.0）

卧龙决策支持系统是一款围绕“**信息整理—认知审计—因果推演—系统演化—行动建议**”展开的决策认知增强产品。
系统从多源公开信息中提取事实，识别偏见和隐藏前提，构建可解释的因果关系与演化路径，最终输出以“**主要矛盾 + 临界预警 + 最小行动实验**”为核心的决策白皮书。
# 核心价值

1. **降低噪音密度**：从大量经验贴、软广、情绪帖中提取真正有决策价值的条件信息。
2. **强制慢思考**：把“观点”拆成“事实—证据—假设—风险”，压制情绪或错误认知驱动决策。
3. **提升解释性**：任何结论都必须可追溯到证据片段，用户可一键查看相关原文。
4. **支持试错而非豪赌**：输出小步可验证的 MVP 行动，而不是鼓励单次高风险押注。
5. **在证据不足时主动止损**：样本量不足或证据冲突过大时，系统必须拒绝深度推演。


## 核心业务原则

1. **证据先于判断**：没有证据绑定的内容必须明确标识。
2. **事实与推断分层展示**：界面和数据结构都要明确区分“事实 / 观点，并标注相应存在的逻辑错误或事实错误以及隐藏信息”。
3. **长尾条件优先保留**：与主流结论相反但带有关键边界条件的信息，不可被简单去重吞没。
4. **反证必须出现**：每条主流判断旁必须尽可能展示反面风险或例外条件。
5. **不足即降级**：证据不够时，只展示信息池和审计结果，不做因果图和世界线。
6. **行动建议以低成本实验为核心**：输出以现实世界反馈为目标，而非抽象鼓舞性建议。

## 用户输入模型
为保证推演质量，用户输入决策情景，用于检索相关内容，基于大模型的对用户输入进行搜索关键词转换


##  端到端用户旅程

1. 用户输入决策情景创建决策案例。
2. 用户选择信息源、样本上限，由大模型生成搜索关键词【关键词可带占位符由用户补充信息，比如毕业年份，可生产多条，生成后的关键词要支编辑】。
3. 系统完成采集【针对每个关键词进行信息爬取】、清洗、去重与融合【可批量借助大模型对采集的信息进行处理】形成信息池。
4. 基于信息池，由大模型对收集到的信息进行提取凝练与梳理，构建以‘事实论点’为核心的事实论点审查台，用户在“事实论点审查台”查看事实论点卡片和相应的证据以及偏见和隐藏事实等在内等错误认知点。
5. 基于凝练的事实论点，借助大模型构建因果沙盘，用户进入“因果沙盘”调节变量，进行反事实推演【可借助大模型实现】。
6. 世界线推演先具有搜集到的事实论点和因果，让用户补充信息，基于用户的信息和构建的事实和因果，基于构建的事实和因果以及用户自身情况展开世界线推演【借助大模型】，可对未来节点进行干预，以改变世界线【借助大模型进行演化】。
7. 基于事实，因果，世界线，生成白皮书。用户阅读“破局白皮书”，获得主要矛盾、临界点和本周可执行动作。
8. 用户对事实、推演、建议进行反馈，形成后续优化闭环。

## 功能需求清单

| 编号 | 功能模块 | 关键要求 | 关键输出 | 验收标准 |
| --- | --- | --- | --- | --- |
| **FR-01** | 决策建模与案例创建 | 支持用户输入决策情景，系统解析并创建初始案例 | Case Profile | 对用户的输入可酌情进行情景特征提取与关键词转化，支持对关键词进行编辑 |
| **FR-02** | **信息源配置与搜商** | **用户选择信息源与样本量；大模型基于决策情景自动生成并推荐搜索关键词** | Search Strategy & Source Plan | **系统需展示大模型生成的搜索词，支持用户修改**；可见数据源分布、采集进度 |
| **FR-03** | 数据采集与过滤 | 区分有效经验、情绪发泄、广告/软广、重复转发（可借助大模型批量处理） | Source Pool | 可查看被过滤内容数量与原因分类；**爬虫行为需合规** |
| **FR-04** | 去重与融合提炼 | 先做近重复聚类，再由大模型保留边界条件进行融合与凝练 | Fact Cards / Fact Atoms | 每个原子事实必须绑定证据片段；不能因去重丢失关键条件差异 |
| **FR-05** | 事实审查与认知审计 | 借助大模型提取事实论点，标记潜在偏见、隐藏前提、利益相关及错误认知点 | Audit Alerts | 每张事实卡必须区分“事实/推断/风险/假设”及认知偏差提示 |
| **FR-06** | 一键溯源 | 点击事实论论点卡片可展开原文链接 | Evidence Panel | 每个事实论点都能回到至少 1 个原始证据 |
| **FR-07** | 因果沙盘构建 | 借助大模型生成可交互 DAG，展示事实间的变量、方向、因果解释 | Causal Graph | 图必须可解释、可校验；存在逻辑冲突时需显式提示 |
| **FR-08** | **反事实推演与模拟** | **支持用户在沙盘中调节变量状态进行反事实推演，返回下游影响、原因和风险** | Simulation Result | 输出必须包含受影响节点、影响方向、依据事实、逻辑警告 |
| **FR-09** | **世界线演化与干预** | **引入用户自身情况与补充信息，结合因果沙盘推演不同时间线（6个月/1年/3年）；支持用户干预未来节点** | Worldline Tree & Intervention Log | 每条路径需给出触发条件与不确定性；**必须支持接收用户临时补充的约束条件并重算路径** |
| **FR-10** | 破局白皮书 | 基于事实、因果与世界线，输出主要矛盾、临界预警、关键未知数、本周 MVP 行动 | Executable Whitepaper | 每条行动建议都要说明验证目标、预期反馈、成本与时间 |
| **FR-11** | 证据不足降级拦截 | 当样本不足或冲突过大时，禁止系统继续深推 | Insufficient Evidence Notice | 系统必须说明“不足原因 + 建议补充什么信息” |
| **FR-12** | 闭环反馈与纠错 | 用户可对事实、推演、建议标记错误或低价值 | Feedback Records | 被标记内容进入审查队列，影响后续信息源权重与模型评估 |

## 五大交互视图定义

| 视图 | 展示重点 | 核心交互 | 空态/异常态要求 |
| --- | --- | --- | --- |
| **1. 信息池与融合概览** | 采集总量、来源分布、搜索词策略、过滤与融合结果 | 修改来源与样本量、**微调大模型生成的搜索词**、查看过滤分类 | 若采集失败或来源不足，必须明确说明原因并建议更换搜索词 |
| **2. 事实论点审查台** | 事实论点、底层证据、偏见预警、隐藏事实、认知谬误 | 点击卡片溯源、切换“事实/风险/假设”视图、反馈纠错 | 若某条仅为推断或存在强偏见，必须显式高亮标记 |
| **3. 可交互因果沙盘** | 变量节点、因果边关系、混淆因子、对撞因子 | 拖拽或调整变量状态、**触发反事实推演**、查看影响链路 | 若图不满足稳定推演条件，禁止启动模拟并提示缺失的因果链 |
| **4. 系统演化世界线** | 不同路径在不同时间节点（如6个月/1年/3年）的分支演化 | **输入自身实际情况、补充缺失信息、对未来节点施加干预**、切换时间轴 | 不得输出伪精确概率；只展示置信等级或区间；信息不足时提示补充 |
| **5. 破局白皮书** | 核心主要矛盾、转化临界点、本周 MVP 可执行动作 | 导出、分享、阅读反馈、勾选已完成行动清单 | 若核心未知数过多，白皮书中必须优先列出“待验证未知项” |
---
*(注：因果沙盘视图的复杂性较高，前端设计时需确保有向无环图的层级清晰，避免视觉混乱)*

### 修正后的业务规则 (Business Rules)


| 编号 | 规则 |
| --- | --- |
| **BR-01** | **证据充分性门槛**：默认要求有效源文档不少于 20 篇、每个候选路径至少有 5 条 grounded facts；阈值可根据情景动态配置。 |
| **BR-02** | **证据绑定强制规则**：最终白皮书与审查台中的每一条事实句，都必须严格绑定底层信息池的证据片段 ID。 |
| **BR-03** | **认识论标签强制规则**：所有大模型提炼的输出都要标记为“事实 / 推断 / 风险提醒 / 假设”之一，绝不混淆。 |
| **BR-04** | **概率表达限制**：若未经过真实数据校准，不输出类似“37%”这类精确数字概率，仅输出高/中/低置信度或概率区间。 |
| **BR-05** | **降级优先于编造 (Hallucination Control)**：当因果链断裂、冲突证据过多或关键变量缺失时，系统必须停止深度世界线推演，转为请求用户补充信息。 |
| **BR-06** | **反证优先展示**：对于热点结论或单一推断，旁边必须尽量附带反例、隐藏事实、偏见提示或适用边界。 |
| **BR-07** | **用户纠错降权**：用户标记为错误的事实或低价值内容，立刻进入人工或高级模型审查队列，并自动降低该数据源后续的采信权重。 |
| **BR-08** | **合规采集原则**：所有数据爬取与接入必须遵守来源平台协议（Robots.txt）、授权边界和隐私数据留存策略。 |
| **BR-09** | **用户语境与隐私隔离 (新增)**：用户在世界线推演中补充的“自身情况”数据，仅限本次 Session 推演使用，不得混入公共事实信息池或用于未经脱敏的模型微调。 |
| **BR-10** | **搜索词收敛规则 (新增)**：大模型基于决策情景生成的搜索关键词，必须包含至少一个限制性定语，防止出现过于泛化的爬虫指令导致计算资源浪费。 |                                  |

### 修正后的上线验收标准 (Acceptance Criteria)

主要修正点：补充了搜索词生成、反事实推演交互以及世界线干预的验收条件。

满足以下条件方可进入上线评审：

1. **信息源阶段**：系统能基于用户输入的决策情景，成功由大模型生成并展示合理的搜索关键词。
2. **溯源能力**：审查台内的任意事实论点卡片、白皮书事实句，均可一键回到并高亮原始证据片段。
3. **安全拦截**：证据不足或逻辑冲突严重的案例会被正确拦截，明确提示用户补充具体信息，不强行产生因果图和世界线。
4. **沙盘推演**：因果图无非法闭环（或被显式标记为“未确认”）；**用户能成功调节沙盘变量并得出合理的反事实推演结果**。
5. **世界线演化**：世界线输出包含基准线、理想线、恶化线；**系统能成功接收用户补充的个人情况，并允许用户干预未来节点以改变最终路径**。
6. **行动建议**：白皮书输出的所有动作建议均符合“本周可执行、低成本、可验证”的 MVP 标准。
7. **异常处理**：模型调用失败、爬虫源失败、解析失败时，前端能看到明确的状态反馈与补救按钮。
8. **后台追踪**：关键的演化日志、API 调用成本、模型版本及用户纠错记录，均可在后台管理面板中完整追踪。

*(注：系统演化世界线本质上是对抗不确定性的决策树，前端在展示基准/理想/恶化线时，应强化路径分叉点的触发条件提示)*
