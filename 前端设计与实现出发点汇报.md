# 卧龙项目：前端设计与实现出发点汇报（基于当前代码）

## 1. 汇报目的
- 盘点当前前端已实现内容，明确每个页面“展示什么信息、如何展示”。
- 为前端设计与实现人员提供统一的现状基线与后续迭代出发点。

## 2. 当前前端总体设计基线

### 2.1 技术栈与工程形态
- Vue 3 + TypeScript + Vite
- Vue Router（流程路由）
- Pinia（状态管理）
- Element Plus（基础组件）
- Tailwind CSS（布局与视觉快速构建）
- ECharts + vue-echarts（图谱与时间树可视化）

### 2.2 产品结构（信息架构）
- 首页入口：`/case/create`
- 工作流主容器：`/case/:id/workflow/*`
- 6 步主流程：
  1. 信息采集（strategy / collection）
  2. 信息汇聚池（summary / facts）
  3. 事实审查（audit）
  4. 因果沙盘（sandbox）
  5. 世界线演化（worldline）
  6. 破局白皮书（whitepaper）
- 独立功能页：
  - 大模型配置：`/settings`
  - 历史档案：`/history`

### 2.3 视觉与交互语言
- 视觉基调：浅色背景（slate 系）+ 蓝色主色（primary）。
- UI 组织：卡片化、标签化、分区明显。
- 状态表达：Loading/Empty/Error/Success 四态覆盖较完整。
- 动效策略：以轻量过渡（fade、pulse、ripple）强化状态变化反馈。
- 可视化策略：
  - 因果关系：Graph（DAG）
  - 时间推演：Tree（世界线分支）

## 3. 页面级盘点（展示信息 + 展示方式）

## 3.1 创建案例页 `/case/create`
**展示信息**
- 用户决策情景文本输入
- 历史档案、模型配置快捷入口

**展示方式**
- 中央单卡表单（大文本输入 + 主按钮）
- 顶部右侧轻量导航按钮
- 辅助能力标签（自由输入 / AI 解析 / 多维推演）

**核心交互**
- 输入描述后创建案例，成功跳转 workflow。

---

## 3.2 工作流容器页 `/case/:id/workflow/*`
**展示信息**
- 当前案例 ID
- 全流程步骤与当前进度位置

**展示方式**
- 顶部粘性 Header + 横向步骤条
- 主内容区通过子路由承载

**核心交互**
- 任意步骤点击跳转
- 历史、设置、返回首页快捷入口

---

## 3.3 策略配置页 `/strategy`
**展示信息**
- AI 自动生成关键词
- 采集平台配置（平台、额度、评论开关/上限）
- 分块提取参数（chunk size/overlap/token）

**展示方式**
- 双段流程区块：关键词编辑 + 信息源配置
- 底部固定操作条（开始采集）

**核心交互**
- 关键词增删改
- 平台启用与参数配置
- 启动采集任务并进入采集进度页

---

## 3.4 采集进度页 `/collection`
**展示信息**
- 爬取进度、过滤进度、当前平台
- 提炼成果计数（高价值情报条数）

**展示方式**
- 中心环形进度视觉（动态百分比 + 状态文案）
- 成果提示卡（可中途跳转信息池）

**核心交互**
- 轮询刷新状态
- 完成后自动跳转信息池总览

---

## 3.5 信息池总览页 `/summary`
**展示信息**
- 总抓取量、过滤量、有效信源量
- 平台抓取上限、评论抓取开关、分块参数
- 执行对账结果（配置与实际执行一致性）

**展示方式**
- 左侧“质量分布统计卡”
- 右侧“下一阶段工作流提示卡”
- 样本不足时顶部告警（WarningAlert）

**核心交互**
- 样本不足回退策略页
- 进入事实审计页

---

## 3.6 信息池卡片页 `/facts`
**展示信息**
- 每条事实摘要内容
- 来源平台与原帖链接
- 事实类型标签、引文计数、审计提示、反面证据

**展示方式**
- 瀑布流/卡片墙布局
- 卡片头部来源标签，卡片尾部来源下拉菜单
- 长文本折叠与渐隐遮罩

**核心交互**
- 展开/收起内容
- 打开原帖链接
- 跳转审计台

---

## 3.7 事实审计页 `/audit`
**展示信息**
- 事实/推论/假设的分类计数
- 事实卡详情、风险提示、偏差提醒
- 证据链明细、思维链结构化下钻

**展示方式**
- 事实卡列表（FactCard）
- 右侧抽屉证据溯源（SourceTraceDrawer）
- 弹窗展示思维链与中间结构

**核心交互**
- 标记“有价值 / 低价值”
- 按引文下钻原始证据
- 进入因果沙盘

---

## 3.8 因果沙盘页 `/sandbox`
**展示信息**
- 因果图节点与边（含正负关系）
- 节点拓扑提示（混杂因子/对撞因子）
- 推演结果、风险警告、操作快照历史

**展示方式**
- 左侧图谱（Graph）+ 右侧控制台（SimulationPanel）
- 顶部显示反事实洞察/拓扑提示标签
- 干预后高亮涟漪与结果横幅

**核心交互**
- 点选节点、数值干预、文本 What-if 干预
- 撤销回退、重铸图谱
- 进入世界线演化

---

## 3.9 世界线演化页 `/worldline`
**展示信息**
- 多分支路径（baseline/best/worst/custom）
- 分阶段节点描述与触发条件
- 主导回路、新主要矛盾、世界线摘要
- 个人语境探针问题与回答

**展示方式**
- 左侧时间树（TimelineTree）
- 右侧文本干预面板（WorldlineInterventionPanel）
- 无语境时使用模态（ContextInjectionModal）引导输入

**核心交互**
- 注入语境并生成世界线
- 文本扰动新增分支
- 撤销干预、重铸推演
- 跳转白皮书生成

---

## 3.10 白皮书页 `/whitepaper`
**展示信息**
- 核心结构性矛盾、临界风险预警
- 矛盾转化路径
- 核心未知数池
- MVP 行动建议与纠错反馈
- 推理链原始结构

**展示方式**
- 文档式长页排版（可导出）
- 分章节卡片展示
- 顶部操作栏 + 思维链弹窗

**核心交互**
- 动作建议勾选与反馈
- 查看推理链
- 导出 PDF（html2canvas + jsPDF）

---

## 3.11 设置页 `/settings`
**展示信息**
- LLM Provider 列表（name/baseUrl/model/key 掩码）
- 阶段绑定关系（collection/audit/sandbox/worldline）

**展示方式**
- 上部表格管理服务商
- 下部 4 卡片绑定阶段
- 新增/编辑服务商弹窗

**核心交互**
- Provider 增删改
- 连通性测试
- 阶段模型路由绑定

---

## 3.12 历史页 `/history`
**展示信息**
- 历史案例标题、创建时间、状态、标签

**展示方式**
- 卡片网格展示（hover 显示操作区）

**核心交互**
- 恢复到对应流程步骤
- 删除历史案例

## 4. 当前前端展示模式总结（可作为设计系统抽象）
- 表单输入类：场景创建、语境注入、文本干预。
- 卡片列表类：信息池、事实审计、历史案例、行动清单。
- 状态看板类：采集进度、信息池统计、全局步骤进度。
- 图谱可视化类：因果 DAG。
- 时间分支可视化类：世界线 Tree。
- 文档输出类：白皮书阅读与导出。

---

## 5. 现阶段实现特点与约束

### 5.1 优势
- 主流程完整闭环已具备（从输入问题到输出白皮书）。
- 页面状态反馈充分，用户可感知“正在发生什么”。
- 证据链与推演链条具备可追溯能力（抽屉 + dialog）。
- 配置驱动能力有基础（模型路由、阶段绑定）。

### 5.2 当前约束
- 部分阶段信息有重叠（`summary` 与 `facts` 的职责边界可进一步清晰化）。
- 复杂页面更偏桌面端，移动端策略尚不完整（尤其沙盘/世界线）。
- 若后端字段变化，部分页面仍存在直接字段耦合风险（虽已有 adapter，但未全覆盖）。

---

## 6. 给设计与实现团队的“出发点”建议
1. 固化主流程叙事骨架  
将 6 步 workflow 作为唯一主叙事，所有新增能力优先挂靠到这条链路中，避免形成平行分叉体验。

2. 抽象核心 UI 模块  
优先沉淀三大组件簇：`状态看板`、`证据卡片`、`图谱面板`，统一尺寸、边距、字号层级、颜色语义。

3. 明确信息池双页职责  
建议定义：`summary` 侧重“统计与质量门槛”，`facts` 侧重“逐条内容消费与跳转审计”。

4. 优先补齐响应式策略  
为 `/sandbox`、`/worldline` 设计移动端降级展示（抽屉化、分步化、图文切换）。

5. 以接口契约与适配层为前端稳定核心  
新增字段优先走 `adapters` 归一化，减少 view 层直接依赖后端原始结构。

6. 统一状态表达规范  
跨页面统一 Loading/Empty/Error/Success 文案和视觉模板，提高一致性与可维护性。
